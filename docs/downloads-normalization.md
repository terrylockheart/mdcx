# 下载文件归一化与清理（替代 mvfiles.sh）

本文档说明如何用 mdcx 现有配置与刮削流程，覆盖外部脚本 `mvfiles.sh` 的功能，
**无需** 单独的移动/重命名脚本。直接将 mdcx 的影片目录指向原始下载目录即可，
刮削过程会按命名模板输出规范文件名，并可清理残留文件。

## mvfiles.sh 行为 → mdcx 配置对照

| mvfiles.sh 行为 | mdcx 对应配置 / 机制 |
| --- | --- |
| 仅处理大于 100M 的视频 | `要处理的最小文件大小（MB）`（`Config.file_size`，默认 100.0） |
| 删除上传者标记 / 广告串，如 `@xxx`、`-nyap2p.com`、`[cctv666.com]`、`_U3C3`、`麻豆传媒` | `要从文件名中删除的字符串`（`Config.string`）。已内置常见项；自定义站点标记请按 `站点域名@` 形式自行添加 |
| `-UC` → `-C`、`ch.` → `-C`（中文字幕标记） | 中文字幕识别：`cnword_char`（默认含 `-C.`、`-C-`、`ch.`、`字幕`）+ 无码识别（`mdcx/core/file.py`） |
| `_4k` / `_4ks` / `_4k60fps` → `-4K`（清晰度标记） | 清晰度识别（`mdcx/core/file.py` 的 `4K`/`4KS`/`4K60FPS`）+ 命名字段 `four_k` / `definition`；勾选 `folder_hd` / `file_hd` 以在输出名中追加 `-4K` 后缀 |
| 去除 `(...)` 广告括号但保留 `abc-123` 番号 | 番号识别本身基于番号正则提取，括号内噪声不影响识别；如有干扰串可加入 `Config.string` |
| 扫描并删除残留的小文件 / 空文件夹 | `清理规则`：`clean_ext` / `clean_name` / `clean_contains` / `clean_size` / `clean_ignore_*` + “检查并清理文件”功能（`check_and_clean_files`） |
| 将大文件从 `downloads` 移动到 `out` 目录 | 刮削时按 `成功输出目录` 设置自动移动 / 软链输出，无需单独移动步骤 |

## 使用建议

1. 将影片目录设为原始下载目录（或其父目录）。
2. 在 `字符串过滤` 中补充你下载源特有的上传者 / 广告标记。
3. 开启 `清理规则` 并配置扩展名 / 文件名包含项，刮削后用“检查并清理文件”移除残留。
4. 需要在文件名体现 `-C`（中文）与 `-4K`（清晰度）时，确认中文字幕识别字符与
   `folder_hd` / `file_hd` 已按需开启。

## 说明

`字符串过滤`（`Config.string`）为**纯删除**规则，作用于番号识别，因而影响最终输出命名；
它不支持 `mvfiles.sh` 中 `sed 's|/[^/]*@|/|g'` 那种“删除 `@` 之前所有内容”的通配删除，
故针对带 `@` 的上传者前缀，请按具体 `域名@` 形式逐条添加。
